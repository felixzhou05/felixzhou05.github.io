## Design Context

### Users
**Primary:** Technical recruiters and hiring managers at Toronto tech companies (Cohere, Wealthsimple, Amazon, startups). They're scanning 50+ portfolios per week, usually at 2pm on desktop in a bright office. Time-to-impression: under 10 seconds. They want to see: (1) this person can actually build things, (2) they're thoughtful, (3) they'd be good to work with.

**Secondary:** Engineering leads doing a final check before scheduling an interview. They'll actually read project descriptions and check GitHub links. They care about technical depth and taste.

### Brand Personality
**Voice:** Calm, precise, opinionated.
**Tone:** Confident without being arrogant. Shows engineering taste through the portfolio itself — the site IS the proof of competence. Lets the work speak; doesn't oversell.
**Three words:** Restrained. Intentional. Sharp.

### Aesthetic Direction
**Style:** Luxury / refined minimalism with editorial influence.
**North stars:** Vercel / Linear / Rauno Freiberg (precision, cold confidence, subtle motion) crossed with Pentagram / Studio Feixen (typographic hierarchy, art-directed layouts, editorial rhythm).
**Theme:** Dual (light + dark, system-synced). Both modes must be intentionally designed — not one adapted from the other.
**Anti-references:** Brittany Chiang dev-classic dark (the current portfolio is already in this lane). Generic glassmorphism. Dark mode with neon/cyan/purple glowing accents. Terminal window components. Card grids with identical layouts.

### Design Principles
1. **Precision over decoration** — Every element earns its place. No decorative filler, no ornamental grid backgrounds, no sparklines-as-decoration.
2. **Show, don't tell** — The portfolio's own design quality demonstrates engineering competence. The site IS the resume.
3. **Dual excellence** — Both light and dark themes are first-class citizens, designed independently, not one inverted from the other.
4. **Typography carries hierarchy** — Type is the primary design tool. Size, weight, and spacing create structure — not color boxes or card borders.
5. **Restrained motion** — One well-orchestrated moment (e.g., page load stagger, a single scroll-driven reveal) beats scattered micro-interactions everywhere.
